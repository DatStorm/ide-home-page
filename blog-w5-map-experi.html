Map of experiments blog week 5

Create a map of experiments that you think could be interesting to carry out. The more the better.

About our experiments(), the first two was more about the concept, we tried to set up our first MVP with its first features. Even it’s something that already existing on the market, we wanted to know if our version fulfils client needs.

About the categorisation, we wanted to experiment with this early because it’s something important to us, we want to distinguish ourselves from other products with this feature. We made a prototype of a visualisation of a detailed categorisation (and sub-categorisation) of our own Storebox data and showed it to multiple people to get their feedback. The responses can be seen in . As we can see in the results, for the majority it was something interesting and different, but they don’t really know if they would use it. We think it’s a normal reaction because it’s something that don’t already exist, so they can’t really know if they will use it.

The next ones are about a new feature we tried to add to our MVP named Track expenses. It’s a feature that allows users to make a custom saving plan to save money. Our first version was functional, but a bit simple, so we tried to add more precision and an auto-generation feature on version 2. It was something that people appreciated.
